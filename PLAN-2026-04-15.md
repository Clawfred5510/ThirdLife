# ThirdLife — Actionable Plan (2026-04-15)

## Context

The current build is broken (`@gamestu/shared` doesn't emit `dist/`, so typecheck + build fail). The domain `thirdlife.xyz` is a GoDaddy parking page — nothing is deployed. The existing "studio dashboard" is an agent-ops console, not a way to play the game, so there is currently no reliable path for the owner to actually test what the bot has built.

The bot has also implemented jobs, NPCs, tutorials, and partial economy systems that are **not wanted for v1**. v1 is deliberately minimal: a city grid of parcels where players claim a parcel and place a business on it. Nothing else.

A sibling site at `thirdlifeworld.xyz` reportedly already has a parcel-claim system whose rules should be mirrored for claim pricing (confirm exact rules before Phase C implementation).

## Decisions

| Decision | Choice |
|---|---|
| Server host | **Railway** (persistent process, WebSockets, volume for SQLite) |
| Client host | **Cloudflare Pages** (static, free CDN) |
| Grid size | **50 × 50 = 2,500 parcels** |
| Claim cost | **Credits-based** — mirror `thirdlifeworld.xyz` rules (interim placeholder: 100 credits; 500-credit starting stipend) |
| Legacy systems (jobs/NPCs/tutorial) | **Feature-flag off**, don't delete |

## Phase B — Strip to the Core Loop (1 session)

### B1. Feature-flag system
- New file `packages/shared/src/features.ts` — flags from env:
  - `FEATURE_JOBS` (default **false**)
  - `FEATURE_NPCS` (default **false**)
  - `FEATURE_TUTORIAL` (default **false**)
  - `FEATURE_DAY_NIGHT` (default true)
- Server: `process.env`. Client: `import.meta.env.VITE_FEATURE_*`.

### B2. Gate legacy systems
- Server: wrap `spawnNPCs`, tutorial init, job room handlers in `if (features.jobs/npcs/tutorial)`.
- Client: hide Job Board, NPC rendering, Tutorial overlay when flag off.
- Keep the code; only the entry points gate.

### B3. Replace the world with a uniform grid
- Rewrite `packages/client/src/game/entities/buildings.ts`:
  - Remove district-based `generatePurchasablePlots`.
  - New `generateParcelGrid(50, 50)` — 2,000 × 2,000 world, 40 × 40-unit parcels.
  - Each parcel: `id = gx * 50 + gy`, `grid_x`, `grid_y`, center `x, z`.
  - Grid roads between parcels (~8-unit width).
- Simplify `packages/client/src/game/scenes/MainScene.ts` (~lines 90–233):
  - Remove hardcoded districts, river, bay, named boulevards.
  - Flat ground + grid roads + parcel slots. Keep sky / day-night.
- Drop landmarks for v1 (can re-add as decoration later).

## Phase C — Parcels & Businesses (2–3 sessions)

### C1. DB schema
- `packages/server/src/db/index.ts` — drop `properties`, create:
  ```sql
  CREATE TABLE parcels (
    id INTEGER PRIMARY KEY,           -- gx*50 + gy
    grid_x INTEGER NOT NULL,
    grid_y INTEGER NOT NULL,
    owner_id TEXT,                    -- FK players.id, null = unclaimed
    business_name TEXT,
    business_type TEXT,               -- free text for v1
    color TEXT,                       -- hex
    height REAL DEFAULT 4,
    claimed_at TEXT,
    FOREIGN KEY (owner_id) REFERENCES players(id)
  );
  ```
- On first boot, INSERT 2,500 empty rows (`gx` 0–49, `gy` 0–49).

### C2. Server messages (Colyseus)
- In `packages/server/src/rooms/GameRoom.ts`:
  - `claim_parcel { parcelId }` → verify unclaimed, deduct credits, set owner, broadcast.
  - `update_business { parcelId, name, type, color, height }` → verify owner, update, broadcast.
- State schema: add a `parcels` map, synced to all clients.
- Player movement sync unchanged.

### C3. Client UI — `ParcelPanel`
- New `packages/client/src/ui/components/ParcelPanel.tsx`:
  - Unclaimed → "Claim this parcel (X credits)" button.
  - Owned by self → edit `name / type / color / height` → `update_business`.
  - Owned by another → read-only owner + business info.
- `MainScene`: pointer raycast → identify parcel by `id` → open panel.
- Minimap: claimed (filled) vs unclaimed (outlined).

### C4. Parcel rendering
- Empty parcel: tinted square on the ground.
- Owned parcel: tinted square + colored box at `color`/`height`, floating label with `business_name`.
- Re-render on Colyseus state changes (dispose + spawn mesh).
- Labels via `AdvancedDynamicTexture`.

### C5. Sanity test
- Claim parcel → set business → refresh → still there.
- Restart server → still there.
- Second client → sees all parcels.

## Phase D — Deploy (1–2 sessions)

### D1. Server prep
- `Dockerfile` at repo root (Node 20-slim, build shared+server, `CMD node packages/server/dist/index.js`).
- `packages/server/src/config.ts` — read `PORT`, `DATABASE_PATH`, `CLIENT_ORIGIN` from env.
- Bind to `0.0.0.0:$PORT`. Verify `GET /health`.

### D2. Railway
- `railway init` → `railway up`.
- Volume at `/data`; `DATABASE_PATH=/data/thirdlife.db`.
- Env: `CLIENT_ORIGIN=https://thirdlife.xyz`, all `FEATURE_*` off.
- Enable WebSockets. Optional custom subdomain `server.thirdlife.xyz`.

### D3. Client prep
- `packages/client/src/network/Client.ts` (~lines 4–8) — keep `window.__GAME_SERVER_URL__` hook; also honor `import.meta.env.VITE_SERVER_URL` as fallback before host-based default.
- `packages/client/.env.production` → `VITE_SERVER_URL=wss://server.thirdlife.xyz`.

### D4. Cloudflare Pages
- Connect GitHub repo.
- Build: `npm -w packages/shared run build && npm -w packages/client run build`.
- Output: `packages/client/dist`.
- Custom domain `thirdlife.xyz` (remove GoDaddy parking — owner action).
- Set `VITE_SERVER_URL` in Pages env.

### D5. `.github/workflows/deploy.yml`
- Rely on each platform's git auto-deploy on push to `master`.
- Keep workflow minimal: typecheck + lint + build on PRs.

### D6. Smoke test
- Owner opens `https://thirdlife.xyz` on a phone on cellular → spawns, moves, chats.
- Second tab → sees first player.
- Claim parcel → reload → still claimed.
